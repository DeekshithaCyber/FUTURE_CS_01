# Future CS 01 
## Objective
Conducted a basic vulnerability assessment on a publicly available demo web application using passive reconnaissance and security analysis techniques.

## Target Website
https://demo.testfire.net/

## Assessment Scope
- Performed only non-intrusive testing
- Analyzed HTTP security headers
- Reviewed website configuration and exposed information
- No exploitation, brute force, or active attacks were carried out

## Tools & Technologies Used
- Nmap
- OWASP ZAP
- Browser Developer Tools (Chrome/Firefox)

## Work Completed
- Collected basic information about the target website
- Checked for missing or weak security headers
- Observed open ports and running services
- Reviewed potential security misconfigurations using passive scanning methods
- Documented findings with screenshots and scan results

## Deliverables
- Detailed Vulnerability Assessment Report
- Screenshots of scan results and observations
- Output files generated from the tools used during testing

## Note
This assessment was performed strictly for educational and learning purposes on a legal demo website designed for cybersecurity practice.
